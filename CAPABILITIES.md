# 灵造 Skill 能力说明（免费版）

灵造是一个**免费、纯方法论**的创作者内容 Skill，给 AI Agent（Claude Code / Codex 等）用于小红书、抖音、公众号的自媒体运营。

**不调用任何付费接口、不需要 API Key、不扣积分**——装上即用。安装方式见 [README.md](README.md)。

> **数据来源规则**：本 Skill 不从付费后端取数据。需要素材时——① 优先用你给的（笔记链接 / 截图 / 文案 / 后台数据 / 草稿）；② Agent 自带 WebSearch 抓公开内容（小红书 / 抖音强反爬，结果可能不全）；③ 真要实时结构化数据 / 逐字稿 / 直接出图的，诚实说「不支持」，不伪造。

**[SKILL.md](SKILL.md)**

## 二、找方向与选题

**[beginner-account-start-and-topic-radar.md](playbooks/beginner-account-start-and-topic-radar.md)**
0→1 冷启动专用。解决「刚开始做、不知道发什么」，给选题发现、关键词树、低粉爆款参考。

**[keyword-insight-report-template.md](playbooks/keyword-insight-report-template.md)**
围绕一个主关键词做洞察报告。按范围分快速 / 标准 / 深度三档，先列清楚要搜哪些词、读哪些素材，你确认后再展开。

**[keyword-to-publishable-content-package.md](playbooks/keyword-to-publishable-content-package.md)**
把一个关键词或模糊话题，直接变成可发的小红书内容包：标题、封面文案、图文结构、口播稿、Vlog 分镜、正文方向、发布关键词，一次给齐。

**[track-difficulty-judgment-library.md](playbooks/track-difficulty-judgment-library.md)**
判断赛道难度和起号门槛——女性成长 / 职场 / 好物 / 本地生活 / 健康 / 时尚 / AI 工具等常见赛道各有结论。

**[monetization-path-judgment-library.md](playbooks/monetization-path-judgment-library.md)**
判断一个赛道或账号能不能变现、适合走哪条路：广告 / 课程 / 社群 / 咨询 / 线索 / 产品 / 店铺 / 企业转化。

## 三、账号研究与对标

**[atian-creator-judgment-framework.md](playbooks/atian-creator-judgment-framework.md)**
A Tian 的账号判断框架：看账号阶段、记忆锚点、内容主线、当前瓶颈，给出方向性结论。

**[creator-case-general-analysis-framework.md](playbooks/creator-case-general-analysis-framework.md)**
通用创作者案例拆解框架：人设原型、记忆点、新叙事、证明体系、受众欲望、内容引擎、评论区需求、商业入口、隐藏资源、可学部分、不可照抄部分、用户匹配度测试。

**[benchmark-account-discovery-quality-gate.md](playbooks/benchmark-account-discovery-quality-gate.md)**
找 / 评判对标账号。自带质量门（还在更新、近期有爆款、赛道匹配、阶段匹配、有可学之处）；默认一轮最多给 5 个，附带主页链接、粉丝量、近期爆款，按你的反馈再扩展。

**[comparable-account-breakdown-report-template.md](playbooks/comparable-account-breakdown-report-template.md)**
判断某一个具体账号值不值得学、能学什么、什么不能照抄（脸、资源、团队、城市、人设等）。

**[self-account-diagnosis-report-template.md](playbooks/self-account-diagnosis-report-template.md)**
诊断你自己的账号。给结构化报告 + 结论卡 + 行动建议，并用「人情味」收尾——把尖锐诊断变成一个能立刻做的小实验，而不是冷冰冰的待办列表。

**[self-account-peer-horizontal-diagnosis.md](playbooks/self-account-peer-horizontal-diagnosis.md)**
横向对比：把你和同赛道、同阶段、同粉丝量级的账号摆在一起比。说「横向对比」「同级账号」「找 5-15w 粉账号和我比」时触发；只是泛泛问「看看我的问题」则走上面的自诊 playbook。

## 四、单条笔记拆解

**[single-note-breakdown-workflow.md](playbooks/single-note-breakdown-workflow.md)**
深拆一条小红书 / 抖音笔记：标题为什么有人点、封面为什么有人停、大纲 / 脚本结构、拍摄与剪辑手法（可见时）、评论区真实需求、爆点机制、可学之处、不可照抄之处，最后改成你自己的图文 / 口播 / Vlog / 知识卡版本。说「完整拆」「深度拆解」「拍摄手法」「分镜」「剪辑节奏」会进入更深一层。

## 五、标题、人设与受众

**[audience-persona-fit-check.md](playbooks/audience-persona-fit-check.md)**
在写标题、配关键词、做内容包之前，先判断这条到底给谁看、谁会点、谁大概率不会点，并据此选受众 / 城市 / 人生阶段关键词。

**[xhs-title-design-check.md](playbooks/xhs-title-design-check.md)**
小红书标题设计与诊断。默认只给 3 个最强标题，每个带关键词锚点和点击理由；除非你明确要，不堆 10 个标题池。

**[xhs-profile-bio-design.md](playbooks/xhs-profile-bio-design.md)**
写或诊断小红书 100 字简介和主页定位——讲清账号给谁看、分享什么、为什么值得关注，并与昵称、置顶笔记、阶段、受众、城市、轻度商业化承接对齐。

## 六、内容创作与分发

**[draft-rewrite-and-benchmark-workflow.md](playbooks/draft-rewrite-and-benchmark-workflow.md)**
改写你的草稿、套用爆款公式、一次复盘多条内容点子。不是只把句子改顺，而是先判断问题、再给可用版本、最后建立回访闭环。

**[mother-content-cross-platform-distribution.md](playbooks/mother-content-cross-platform-distribution.md)**
「一条龙」跨平台分发包。一个选题 / 草稿 / 拆解 / 产品更新，先给小红书 + 朋友圈 + 公众号基础包，再按需扩展到播客、X、知识星球、B 站、视频号 / 抖音、小红书图文包、知识库 SOP。说「一条龙」「全平台同步」「分发包」时触发。

**[reference-image-graphic-note-workflow.md](playbooks/reference-image-graphic-note-workflow.md)**
根据你给的参考图，产出小红书 4 页或 7 页图文包：每页标题、页面文字、版式说明、图片 prompt、正文文案、评论区引导。

## 七、视觉、封面与出图（prompt-only，不出图）

> 免费版不内置出图接口。这一组产出的是**可直接粘贴到 Midjourney / SD / GPT-4o 等工具的 prompt + 视觉 brief**，由你自己生成图片，Agent 不调用任何出图命令。

**[visual-generation-and-cover-workflow.md](playbooks/visual-generation-and-cover-workflow.md)**
视觉路由。把封面、图文、公众号配图、无人知识卡、产品 / 电商视觉，分别路由到对应风格，给出可直接用的 prompt 包方向。

**[image-generation-execution-workflow.md](playbooks/image-generation-execution-workflow.md)**
出图执行。产出可粘贴的 prompt + 视觉 brief，并跑一遍「视觉导演」质检：图丑、信息太挤、像模板、风格不对时，给出具体返修 brief，而不是把原始 prompt 丢给你自己调。

**[image-generation-agent-integration-guide.md](playbooks/image-generation-agent-integration-guide.md)**
模型无关的出图规范：好图 vs 烂图标准、参考图怎么用、已知生成坑、友好失败处理。

**[visual-reference-style-library.md](playbooks/visual-reference-style-library.md)**
A Tian 视觉风格库分类：旅行美食封面、公众号配图、AI 人物信息图、灵造无人知识卡、产品转化图、人脸关键词视频封面、互动 prompt 封面、截图密集型图文、房间即人设生活方式封面。

## 八、发布与复盘

**[pre-publish-readiness-check.md](playbooks/pre-publish-readiness-check.md)**
发布前就绪检查。先确认内容是否真做完，再逐项查：内容清晰度、图 / 页齐不齐、封面识别度、标题点击力、前 3 行（或前 3 秒）、关键词是否自然埋入。

**[publishing-keyword-design-check.md](playbooks/publishing-keyword-design-check.md)**
给成品定最终 10 个小红书发布关键词，并检查标题、封面文案、开头几行、关键词位有没有自然带上这些词。

**[post-publish-data-review-workflow.md](playbooks/post-publish-data-review-workflow.md)**
已发布笔记复盘。你把笔记链接 + 后台截图 + 脚本 + 封面发来，按 24h / 48h / 7d 判断到底卡在曝光、点击、读完 / 完播，还是关注转化。

## 九、知识库、跟进与产品判断

**[content-knowledge-base-workflow.md](playbooks/content-knowledge-base-workflow.md)**
把收藏的笔记、公开博主链接、关键词结果、爆款例子、博主拆解，沉淀成你自己的知识库：选题库、标题库、封面库、结构库、账号参考库、博主研究库、发布复盘库。

**[retention-and-follow-up-loop.md](playbooks/retention-and-follow-up-loop.md)**
收尾与跟进。每次输出结尾给一个具体下一步；密集内容提供 Word / 网页预览 / 知识库打包，不让用户的话掉地上；诊断准但缺行动时，转向「激活包」而不是继续施压。

**[product-judgment-and-feedback-loop.md](playbooks/product-judgment-and-feedback-loop.md)**
判断用户真正卡在哪、用人话讲清灵造能做什么、把内容 / 销售叙事搭起来、把反馈转成产品迭代，并区分哪些需求值得做、哪些是噪音。

## 能做与做不到（免费版边界）

**能做的**（方法论完整）：账号诊断、对标筛选与评判、爆款拆解、选题、关键词、标题、封面文案、图文结构、跨平台分发包、发布前检查、数据复盘、知识库沉淀，以及出图 prompt 包。

**做不到的**（需付费后端，会诚实告知，不伪造）：实时结构化抓取小红书 / 抖音数据、保证拿到视频逐字稿、直接生成图片（现改为产 prompt 包，由你自己出图）。

## 怎么触发

在支持 Skill 的 Agent 里说「用灵造…」或 `$lingzao`，Agent 会先读 [SKILL.md](SKILL.md)，再按你的任务进入对应 playbook。常用示例：

- 「用灵造帮我找 5 个持续更新、有爆款的小红书对标账号」
- 「用灵造完整拆一下这条小红书笔记为什么爆」（附笔记链接 / 截图 / 文案）
- 「用灵造拿我的账号和 5-15w 粉 AI 博主横向对比」
- 「用灵造给我做一条女性成长图文内容包」
- 「用灵造检查我的标题、封面、前三行和关键词有没有埋进去」
- 「用灵造根据参考图做一张小红书封面」（产出 prompt 包，你自行出图）
